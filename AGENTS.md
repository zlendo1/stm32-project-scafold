## CONVENTIONS

**Code generation**: STM32CubeMX — edit `.ioc` in CubeMX, regenerate, code
inside `USER CODE BEGIN/END` preserved.

**Formatting**: `.clang-format` (LLVM-based, Allman braces, 80 col, 4-space
indent)

**Clangd**: `.clangd` configured for ARM cross-compile; compile_commands.json
from `build/Debug/`

**Build presets**: `cmake --preset Debug` or `Release` (Ninja generator)

## ANTI-PATTERNS (THIS PROJECT)

- **NEVER** edit `Drivers/` — regenerated by CubeMX
- **NEVER** edit `.ioc` manually — use STM32CubeMX GUI
- **NEVER** edit code outside `USER CODE` blocks in CubeMX-generated files
- **NEVER** use `-fcyclomatic-complexity` flag — not supported by most GCC
  toolchains

## UNIQUE STYLES

- CMake-based build (not Makefile) — unusual for STM32 tutorials
- Clang tooling (clang-format, clangd) — project prefers over GCC-native
- HAL Driver library built as OBJECT library (`STM32_Drivers`)

## HARDWARE

- **Board**: STM32F3DISCOVERY
- **MCU**: STM32F303VCTx (Cortex-M4, FPU)
- **Clock**: 48MHz (HSE 8MHz bypass + PLL x6)
- **Memory**: 256KB FLASH @ 0x08000000, 40KB RAM @ 0x20000000, 8KB CCMRAM @
  0x10000000
- **LEDs**: PE8-PE15 (8 LEDs in compass pattern: LD3-LD10)
- **Button**: PA0 (B1, active high)
- **Peripherals configured**: I2C1, SPI1, USB FS
