# PROJECT KNOWLEDGE BASE

## OVERVIEW

CMake project template for STM32F3DISCOVERY (STM32F303VCTx, Cortex-M4 + FPU).
Provides build system, toolchain configs, dev tooling, and build scripts. User
generates STM32CubeMX code into this directory to create a complete firmware
project.

## STRUCTURE

```
./
├── cmake/
│   ├── gcc-arm-none-eabi.cmake    # ARM GCC toolchain (default)
│   ├── starm-clang.cmake          # STARM Clang toolchain (alternative)
│   └── stm32cubemx/
│       └── CMakeLists.txt         # Placeholder (CubeMX overwrites this)
├── scripts/
│   ├── build.sh                   # Bash build/flash/debug automation
│   ├── build.ps1                  # PowerShell equivalent
│   └── README.md                  # Script usage docs
├── CMakeLists.txt                 # Top-level build config (edit project name here)
├── CMakePresets.json              # Debug/Release presets (Ninja generator)
├── .clang-format                  # Code formatting (LLVM, Allman, 80 col, 4-space)
├── .clangd                        # LSP config for ARM cross-compile
└── .gitignore                     # Ignores Drivers/, build/, IDE files
```

After CubeMX code generation, these directories appear:

```
├── Core/           # User application code (edit here)
│   ├── Inc/        # Headers
│   └── Src/        # Implementation (main.c, interrupts)
├── Drivers/        # ST HAL/CMSIS (gitignored, regenerated by CubeMX)
├── startup_*.s     # Reset handler, vector table
└── STM32*_FLASH.ld # Linker script (memory layout)
```

## WHERE TO LOOK

| Task               | Location                           | Notes                           |
| ------------------ | ---------------------------------- | ------------------------------- |
| Project name       | `CMakeLists.txt`                   | `set(CMAKE_PROJECT_NAME ...)`   |
| Build settings     | `CMakeLists.txt`                   | Add sources, includes, defines  |
| Toolchain flags    | `cmake/gcc-arm-none-eabi.cmake`    | MCU flags, optimization, linker |
| Build presets      | `CMakePresets.json`                | Debug/Release, Ninja generator  |
| CubeMX integration | `cmake/stm32cubemx/CMakeLists.txt` | Auto-generated by CubeMX        |
| Build/flash/debug  | `scripts/`                         | Auto-detects project name       |

## CONVENTIONS

**Code generation**: STM32CubeMX with Toolchain/IDE set to "CMake". Edit `.ioc`
in CubeMX, regenerate. Code inside `USER CODE BEGIN/END` blocks is preserved.

**Formatting**: `.clang-format` (LLVM-based, Allman braces, 80 col, 4-space
indent)

**Clangd**: `.clangd` configured for ARM cross-compile; uses
`build/Debug/compile_commands.json`

**Build presets**: `cmake --preset Debug` or `Release` (Ninja generator)

**Scripts**: Auto-detect project name from `CMakeLists.txt`. Single source of
truth.

## ANTI-PATTERNS

- **NEVER** edit `Drivers/` — regenerated by CubeMX
- **NEVER** edit `.ioc` manually — use STM32CubeMX GUI
- **NEVER** edit code outside `USER CODE` blocks in CubeMX-generated files
- **NEVER** edit `cmake/stm32cubemx/CMakeLists.txt` — CubeMX overwrites it
- **NEVER** use `-fcyclomatic-complexity` flag — unsupported by most GCC
  toolchains

## COMMANDS

```bash
cmake --preset Debug && cmake --build build/Debug    # Build
./scripts/build.sh build                              # Build (via script)
./scripts/build.sh flash                              # Flash via OpenOCD
./scripts/build.sh debug                              # GDB debug session
```

## HARDWARE

- **Board**: STM32F3DISCOVERY
- **MCU**: STM32F303VCTx (Cortex-M4, FPU)
- **Memory**: 256KB FLASH @ 0x08000000, 40KB RAM @ 0x20000000, 8KB CCMRAM @
  0x10000000
- **Toolchain flags**: `-mcpu=cortex-m4 -mfpu=fpv4-sp-d16 -mfloat-abi=hard`
